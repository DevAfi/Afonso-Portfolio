About This Project

A comprehensive, local-first password manager with a beautiful terminal interface built using Rich. Features include master password authentication with secure PBKDF2 hashing, password entry management (add, retrieve, update, delete), advanced password strength analysis with visual feedback, secure password generation with customizable parameters, automatic clipboard integration, and enterprise-grade Fernet encryption for all stored data. This project demonstrates advanced cryptographic principles, secure software development practices, and expertise in building production-ready CLI applications.

Technical Implementation

ğŸ” Security Architecture

The application implements a robust security model based on industry-standard cryptographic practices:

  • Key Derivation: PBKDF2 with SHA-256 and 100,000 iterations
  • Encryption: Fernet (AES-128 + HMAC) for authenticated encryption
  • Salt Generation: Cryptographically secure random salt for each session
  • Password Hashing: Master password never stored in plaintext

First Time Setup

The application automatically detects if no master password exists and will prompt you to create one. The master password must be at least 8 characters long and is securely hashed using PBKDF2.

Lessons Learned & Challenges Faced

  • Key Derivation: Balancing security (high iterations) with performance (user experience) - settled on 100k PBKDF2 iterations
