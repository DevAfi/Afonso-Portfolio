About This Project

GitHub Data Analyzer CLI is a powerful command-line tool that provides comprehensive insights about any public GitHub repository. Built with Python and the Rich library, it offers beautiful terminal output with colored tables, panels, and progress indicators. This tool solves the problem of quickly evaluating repository health, activity patterns, and contributor statistics without navigating through multiple GitHub pages.

  • üìà Commit Analysis: Total commits, commits per day, most active day and hour patterns

‚úÖ Repository Health Score Algorithm

The comprehensive health rating evaluates repositories across three key dimensions:

  • Activity Score (40% weight):
    • 100 points: Updated within last 7 days
    • 80 points: Updated within last 14 days
    • 60 points: Updated within last 30 days
    • 40 points: Updated more than 30 days ago
  • Contributor Score (30% weight):
    • 100 points: 50+ contributors
    • 80 points: 20-49 contributors
    • 60 points: 10-19 contributors
    • 40 points: 5-9 contributors
    • 20 points: Less than 5 contributors
  • Documentation Score (30% weight):
    • 40 points: Has description
    • 30 points: Has license
    • 30 points: Has wiki enabled

Overall Rating: EXCELLENT (80-100), GOOD (60-79), FAIR (40-59), POOR (0-39)

Getting Started

Installation

# Clone the repository
git clone https://github.com/DevAfi/Github-Data-Analyzer.git
cd Github-Data-Analyzer

# Create and activate virtual environment (optional but recommended)
python -m venv venv

# macOS/Linux
source venv/bin/activate

# Windows PowerShell
./venv/Scripts/Activate.ps1

# Install dependencies
pip install -r requirements.txt

GitHub Token Setup (Optional but Recommended)

While the tool works without authentication, using a GitHub personal access token provides:

  • Higher rate limits: 5,000 requests/hour vs 60 requests/hour unauthenticated
  • Better reliability: Less likely to hit rate limits
  • Access to private repos: If you have access
# Create a .env file in the project root
echo "GITHUB_TOKEN_CODE=your_github_personal_access_token_here" > .env

# To get a token:
# 1. Go to GitHub Settings ‚Üí Developer settings ‚Üí Personal access tokens ‚Üí Tokens (classic)
# 2. Generate new token with 'public_repo' scope
# 3. Copy token to .env file

Usage

Interactive Mode: Run without arguments for interactive prompts

python main.py

Command-Line Mode: Analyze a specific repository directly

# Analyze a popular repository
python main.py torvalds/linux

# Analyze a React repository
python main.py facebook/react

# Disable caching for fresh data
python main.py --no-cache microsoft/vscode

üèóÔ∏è Architecture Decisions

API Rate Limits

Without Authentication

  • 60 requests per hour
  • IP-based rate limiting

With Authentication

  • 5,000 requests per hour
  • Token-based rate limiting
  • More reliable for batch operations

Current Usage

  • Each repository analysis requires ~4 API calls (repo, commits, contributors, languages)
  • With caching enabled, repeated analyses use cached data
  • Rate limit remaining is displayed in warnings when below 10 requests
